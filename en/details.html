Integration services (SOAP/HTTP inputs)

See an example of SOAP/HTTP web service interfaces hosted in IBM Integration Bus using integration service projects.
OVERVIEW

IBM Integration Bus provides a specific project type, an integration service, for developing web service implementations. These help you define and implement a web service interface which can be called from other standard clients, and where the interface is defined by WSDL. This tutorial shows a simple example where a web service operation implements a transformation between temperature formats.

The integration service editor separates the tasks of defining an interface from implementing that interface. In this example, the interface defines two operations, both of which have numeric inputs and outputs representing temperature in different scales. The interface is stored as a standard WSDL document, and an editor is provided to show and help you build or change that interface. Each operation is implemented as a subflow, using the built-in Mapping nodes to transform the incoming requests. Error handling is also demonstrated.

You will run the integration service project using the built-in Flow Exerciser. This will show you how the incoming web service request is passed into the operation subflows that you implement. As you run it, you will also see how the integration service is deployed as a single entity, which packages up your web service solution. The WSDL document describing the interface can be retrieved from the service both during development and also after it has been deployed.

This tutorial takes approximately 5 minutes.
