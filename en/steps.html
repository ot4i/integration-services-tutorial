Import projects

On clicking Import, an integration service project called TemperatureConverter will be added to your workspace, and the integration service editor will be opened to show you its contents. Next, click Prepare for an explanation of the different parts of the integration service editor.

Import

Imported projects

In the main pane you will see the integration service editor, which you can reopen by clicking on the integration service Description inside the TemperatureConverter project. The editor has two tabs, one called Service and another called Interface.

If you click on Interface, you will see the two web service operations which have been defined, along with their input and output types. These are called CtoF and FtoC.

If you click back on Service, you will see a link for each operation name defined in the Interface tab, along with another 3 links for error handling (named Timeout, Fault and Catch). Click on each of CtoF and FtoC to see the subflow which implements that operation. There is a Mapping node in each one which performs a transformation on the input data, to convert between Celsius and Fahrenheit temperatures. Double-click the Mapping node to see that transformation, and note that the input and output types for the mapping match what was already defined in the integration service's Interface editor. Click also on the Fault and Catch handlers, which are implemented with another Mapping node in this example to return an error to the external web service client.

Finally, click on the SOAP/HTTP Binding link and look at the Properties pane; this will show you properties which apply to all operations, such as the URL prefix and whether https is enabled.

Next, you will use the Flow exerciser to invoke and send a SOAP request to the integration service. Note the icon in the top-right corner of the editor which you will use in the Run stage.

Follow these steps to see how the Integration Service runs

The TemperatureConverter integration service is shown in the Application Development view of your workspace.

  1. Open the integration service Description of the TemperatureConverter service, and click the Start Flow exerciser icon to start recording the message path through the integration service. Note that this will deploy the integration service to your TESTNODE integration node.
  2. Click the Send Message icon and create a new input message.
  3. Choose the CtoF SOAP operation, edit the message data if you like, and click Send. Your request message is sent to the integration service SOAP input.
  4. Look at the SOAP input and output data shown in the pop-up window.

After you close the Send Message dialog, the message path is highlighted in green on the service description diagram.

  • Click on the highlighted line between the SOAP/HTTP Binding and the operation to see the original input, before it is sent to the operation subflow for processing.
  • Click on the highlighted CtoF operation to expand the message path through the operation subflow.
  • Click on any connection to see the data that passed through that connection. Note the data before and after the Mapping node: the TemperatureInC XML element has changed to a TemperatureInF XML element.

Look in the Integration Nodes view and find your test integration node (called TESTNODE_yourusername). Expand the tree to show the integration servers, then the TemperatureConverter integration service. Click on the Integration Service and see its properties, including the Service URL and Service Query URL. This shows the actual URL where an external client would need to send requests to invoke your service.

Experiment with the operational view of your integration service in the Web administration view. Right-click on your Integration Node (TESTNODE_yourusername) and select Start Web User Interface to open in your browser. Expand the tree as you did earlier, to show your integration server, then its integration services. Click on your integration service and note that the same URLs are shown in the browser as you just saw in your Integration Toolkit.

To stop recording the message path through the flow, click the Return flow to edit mode icon .
